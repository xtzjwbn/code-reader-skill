# Overall Summary Template

Create this file only after:

- pre-reading artifacts are complete
- subsystem writeups are complete
- the final merged repository document is complete

Suggested filename:

- `99-overall-summary.md`

## 1. Repository Purpose

- what the repository appears to do
- what kind of product, platform, service, or tooling it supports

## 2. System Overview

- the major subsystems
- how they relate at a high level
- which subsystems are most critical

## 3. Core Flows

- the most important request, event, batch, or user flows
- which systems participate in each flow

## 4. Key Risks

- the highest-value architectural or operational risks
- important hidden constraints or coupling

## 5. Recommended Reading Order

- where a new engineer should start
- what documents to read next depending on role or goal

## 6. Document Lineage

- which important documents were newly created
- which important documents were updated from existing repository documentation
- which old documents were left unused because the workflow chose rebuild mode
- which systems have important child detail documents for deeper reading

## 7. Open Questions

- the most important unresolved items that still need human confirmation

## Rules

- Keep it shorter than the final merged document.
- Write it as a cross-document guide, not a raw recap.
- Assume the reader may start here before diving into the detailed docs.
